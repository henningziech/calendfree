# Internationalization (i18n) Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add English/German multilanguage support to all frontend pages, with language configurable per-company (public pages) and per-user (dashboard/admin).

**Architecture:** react-i18next with 5 JSON namespaces (common, booking, dashboard, admin, routing). Language stored in DB on User and Company models. A LanguageProvider component uses route matching to determine whether to use the user's or company's language preference.

**Tech Stack:** react-i18next, i18next, date-fns locales (de, enUS)

**Spec:** `docs/superpowers/specs/2026-03-17-i18n.md`

- [ ] **Step 4: Create LanguageProvider**

Create `frontend/src/i18n/LanguageProvider.tsx`:

```tsx
import { useEffect } from 'react';
import { useLocation } from 'react-router';
import { useTranslation } from 'react-i18next';
import { useAuth } from '../context/AuthContext';

/**
 * Determines and sets the active i18n language based on route context.
 * - Dashboard/Admin routes: uses user.language from auth context
 * - Public routes: language is set by the page component itself (from company API data)
 * - Fallback: 'en'
 */
export function LanguageProvider({ children }: { children: React.ReactNode }) {
  const { i18n } = useTranslation();
  const { user } = useAuth();
  const location = useLocation();

  useEffect(() => {
    const isDashboardOrAdmin =
      location.pathname.startsWith('/dashboard') ||
      location.pathname.startsWith('/admin');

    if (isDashboardOrAdmin && user?.language) {
      i18n.changeLanguage(user.language);
    } else if (!isDashboardOrAdmin) {
      // Public pages set their own language via useCompanyLanguage()
      // Default to 'en' until the page component sets it
      i18n.changeLanguage('en');
    }
  }, [location.pathname, user?.language, i18n]);

  return <>{children}</>;
}
```

- [ ] **Step 5: Create date locale utility**

Create `frontend/src/utils/dateLocale.ts`:

```typescript
import { de } from 'date-fns/locale';
import { enUS } from 'date-fns/locale';
import i18n from '../i18n';

/** Returns the date-fns locale matching the current i18n language */
export function getDateLocale() {
  return i18n.language === 'de' ? de : enUS;
}

/**
 * Formats a date string for display, respecting the current language.
 * German: "17. März 2026", English: "March 17, 2026"
 */
export function formatDateLocalized(dateStr: string): string {
  try {
    const date = new Date(dateStr);
    return date.toLocaleDateString(i18n.language === 'de' ? 'de-DE' : 'en-US', {
      day: 'numeric',
      month: 'long',
      year: 'numeric',
    });
  } catch {
    return dateStr;
  }
}
```
